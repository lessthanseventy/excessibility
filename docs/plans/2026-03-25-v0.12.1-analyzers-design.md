# v0.12.1 Analyzer Features Design

Covers GitHub issues #93-#98: six new/enhanced analyzer features for the debug timeline.

## New Enrichers

### `assign_sizes` (replaces `memory`)

Tracks per-assign byte sizes. Deletes the old `memory.ex` enricher.

**Output:**
```elixir
%{
  assign_sizes: %{current_user: 12400, products: 8200, filter: 120},
  total_memory: 20720,
  largest_assign: {:current_user, 12400}
}
```

- Cost: `:expensive`
- Skips Phoenix internals (flash, __changed__, etc.)
- `memory` analyzer updates to read `:total_memory` instead of `:memory_size`

### `ecto_queries` (#95)

Telemetry handler attaching to `[:ecto, :query]` during test execution. Stores queries in an Agent (test-scoped), enricher reads queries that fired between previous and current event timestamps.

**Output:**
```elixir
%{
  ecto_queries: [
    %{source: "products", operation: :select, duration_ms: 1.2, query: "SELECT p0.* FROM..."}
  ],
  ecto_query_count: 2,
  ecto_total_query_ms: 2.0
}
```

- Cost: `:moderate`
- Captures: source table, operation type, duration, query string, repo module

### `push_events` (#98)

Instruments `Phoenix.LiveView.push_event/3` calls via telemetry handler during test. Same Agent pattern as ecto_queries.

**Output:**
```elixir
%{
  push_events: [%{event_name: "update-chart", payload_size: 450}],
  push_event_count: 1
}
```

- Cost: `:moderate`

## New/Modified Analyzers

### `assign_diff` (new, #93)

Detects large assigns re-diffed frequently. Requires: `:assign_sizes`.

- Track per-assign size across events, cross-reference with `changes`
- Warning: >5KB assign in >50% of diffs
- Critical: >20KB assign in >50% of diffs
- Suggestion: extract to LiveComponent or reduce struct size
- Default enabled: true

### `data_growth` (enhanced, #94)

Extends existing analyzer with specific recommendations.

- Append-only growth: suggest `temporary_assigns`
- Large collection with individual updates: suggest Streams
- Add `:suggestion` field to finding metadata
- No new enricher dependencies

### `ecto_query_analysis` (new, replaces `n_plus_one`, #95)

Full Ecto query analysis. Requires: `:ecto_queries`. Deletes old `n_plus_one.ex`.

- Flag events with >5 queries
- N+1 detection: multiple SELECTs on same table in one event
- Slow query detection: individual >100ms or total >500ms per event
- Group findings by source table
- Default enabled: true

### `component_rerender` (new, #96)

Detects unnecessary component re-renders. Requires: `:component_tree`.

- Track component IDs across render events
- Cross-reference with `changes` to find renders without input changes
- Heuristic: stable component count + renders without assign changes = wasted
- Default enabled: false (component_tree is moderate cost)

### `message_flooding` (new, #97)

Detects high-frequency `handle_info` patterns. No enricher needed.

- Sliding window: >10 same-name events within 200ms
- Total count: >20 of any single handle_info type
- Suggest debounce/throttle
- Default enabled: true

### `push_event_volume` (new, #98)

Detects excessive push_event calls. Requires: `:push_events`.

- Flag >5 push_events from single handler
- Flag >3 push_events within 100ms window
- Suggest batching
- Default enabled: false

## Deletions

- `lib/telemetry_capture/enrichers/memory.ex` -> replaced by `assign_sizes.ex`
- `lib/telemetry_capture/analyzers/n_plus_one.ex` -> replaced by `ecto_query_analysis.ex`
- Corresponding test files updated/replaced

## References

All references to `memory_size` in existing analyzers/tests update to `total_memory`. All references to `n_plus_one` analyzer update to `ecto_query_analysis`.
